Project introduction

This project is a smart energy-saving lighting system built using a microcontroller, integrating motion detection, ambient light sensing, and current monitoring to automate and optimize bulb usage.

It connects to Wi-Fi and communicates with Ubidots to publish real-time data such as light status, current consumption, and energy saved, while also allowing remote manual control .The system uses a motion sensor to detect motion, an LDR to measure ambient light, and an current sensor to monitor current flow.

Smart Light and Energy Savings System – Block Diagram

This block diagram illustrates the architecture of a smart lighting system powered by an ESP32 microcontroller. It integrates a Light Dependent Resistor (LDR), motion sensor, and current sensor to automate lighting and track energy usage. Each of the four lamps (9W) is controlled via a switching module, enabling efficient energy management based on real-time environmental inputs.

✓ Light-dependent resistor (LDR): This is a sensor that can tell how bright or dark it is in a room. Think of it like your eyes noticing when it’s getting dark.

✓ Motion sensor: This sensor detects when someone moves in a room, like noticing if a person walks by.

✓ Current sensor: This sensor keeps track of how much electricity is being used, like checking the electricity meter at home.

✓ ESP32 microcontroller: This is the “brain” of the system. It looks at the information from the sensors and decides what to do, like turning lights on or off.

✓ Power supply: This gives electricity to the brain (microcontroller) and the sensors so they can work, like plugging your devices into a wall socket.

✓ Switching mode: This is like a small helper that actually turns each light on or off when the brain tells it to.

✓ Lamps (lamp1, lamp2, lamp3, lamp4): These are the lights that will turn on or off depending on the information from the sensors and instructions from the brain.

Smart Light Control Circuit Diagram

This smart light control circuit is designed to automate lighting based on motion and ambient light conditions. It uses an ESP32 microcontroller to process sensor inputs and control relays that switch 12V lamps on or off.

The system integrates a PIR sensor to detect motion and a Light Dependent Resistor (LDR) to measure brightness. When motion is detected and ambient light is low, the ESP32 activates the corresponding relay to power the lamp.

Each lamp (L1–L4) is controlled via a relay (RL1–RL4) and a transistor (Q1–Q4, BC548) acting as a switch. Diodes (D1–D4) protect the circuit from voltage spikes caused by relay switching. Resistors (R1–R5) ensure proper current flow and protect components.

The circuit also includes an ACS712 current sensor to monitor energy usage, making it suitable for energy-saving applications. The ESP32 receives inputs from sensors and sends control signals through its GPIO pins (D23, D22, D21, etc.).

This setup is ideal for smart home systems, allowing automated lighting that responds to environmental conditions and user presence, improving both convenience and energy efficiency.

1. Global Declarations & Libraries

#include <WiFi.h>
#include <UbidotsEsp32Mqtt.h>
#define PIRPIN             27
#define LDRPIN             34
#define CURRENTSENSORPIN   35
#define RELAY1_PIN         26
#define RELAY2_PIN         25
#define RELAY3_PIN         33
#define RELAY4_PIN         32

const int BULB_WATTAGE = 10;
const int NUM_BULBS = 4;
const int LIGHT_THRESHOLD = 500;

2. Setup Configuration

void setup() {
  Serial.begin(115200);
  
  pinMode(PIRPIN, INPUT);
  pinMode(LDRPIN, INPUT);
  pinMode(CURRENTSENSORPIN, INPUT);
  
  pinMode(RELAY1_PIN, OUTPUT);
  pinMode(RELAY2_PIN, OUTPUT);
  pinMode(RELAY3_PIN, OUTPUT);
  pinMode(RELAY4_PIN, OUTPUT);
  
  digitalWrite(RELAY1_PIN, LOW);
  digitalWrite(RELAY2_PIN, LOW);
  digitalWrite(RELAY3_PIN, LOW);
  digitalWrite(RELAY4_PIN, LOW);
  
  WiFi.begin(WIFI_SSID, WIFI_PASSWORD);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println(" Connected!");
}

3. Main Control Loop

void loop() {
  int motion = digitalRead(PIRPIN);
  int lightLevel = analogRead(LDRPIN);
  float current = readCurrent();
  
  if (forceOn) {
    for (int i = 0; i < 4; i++) {
      if (!bulbOn[i]) {
        controlBulb(i, true);
      }
    }
  } else {
    if (motion == HIGH && lightLevel < LIGHT_THRESHOLD) {
      for (int i = 0; i < 4; i++) {
        if (!bulbOn[i]) {
          controlBulb(i, true);
        }
      }
    } else {
      for (int i = 0; i < 4; i++) {
        if (bulbOn[i]) {
          controlBulb(i, false);
        }
      }
    }
  }
  
  delay(500);
}

4. Custom Functions

void controlBulb(int index, bool state) {
  int pins[] = {RELAY1_PIN, RELAY2_PIN, RELAY3_PIN, RELAY4_PIN};
  digitalWrite(pins[index], state ? HIGH : LOW);
  bulbOn[index] = state;
}

float readCurrent() {
  int raw = analogRead(CURRENTSENSORPIN);
  float voltage = (raw / 4095.0) * 3.3;
  float offset = 3.3 / 2.0;
  float sensitivity = 0.185;
  float current = (voltage - offset) / sensitivity;
  return abs(current);
}

IoT Dashboard & Monitoring

The system leverages MQTT protocol to communicate with Ubidots, enabling real-time monitoring, data visualization, and remote control capabilities. Users can track energy consumption, view system status, and manually override automation from anywhere in the world.

Remote Monitoring – Feature Breakdown

Element of Ubidots Dashboard

✓ Light Status (ON/OFF): Displays the current state of each connected lamp. Helps users know which lights are active at any moment.

✓ Current Consumption: Shows real-time electrical current usage for each lamp. Useful for tracking power draw and identifying high-consumption devices.

✓ Motion Detection: Uses a PIR sensor to detect movement in the monitored area. When motion is detected, the system activates the corresponding light automatically,and user know that the person present to enhancing security and convenience.

✓ Light Detection: Utilizes an LDR (Light Dependent Resistor) to measure ambient light levels. When the surrounding light falls below a certain threshold, the system automatically turns on the lights to maintain visibility and comfort,also display light value in the following gauge.

✓ Manual Control Switches: Allows users to override automation and manually toggle lights on or off from the dashboard interface as he/she wish when no motion detected.
